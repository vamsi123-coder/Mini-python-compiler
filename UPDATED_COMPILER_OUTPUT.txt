PHASE 3 — ICG OUTPUT (THREE REPRESENTATIONS)
================================================================================

1. QUADRUPLES FORMAT (op, arg1, arg2, result)
────────────────────────────────────────────────────────────────────────────

Format: Each instruction has exactly 4 fields
┌──────┬─────────┬──────────┬──────────┬──────────┐
│ Idx  │ Operator│ Arg1     │ Arg2     │ Result   │
├──────┼─────────┼──────────┼──────────┼──────────┤
│ 0    │ =       │ 10       │ -        │ a        │
│ 1    │ =       │ 9        │ -        │ b        │
│ 2    │ +       │ a        │ b        │ t0       │
│ 3    │ +       │ t0       │ 100      │ t1       │
│ 4    │ =       │ t1       │ -        │ c        │
│ 5    │ =       │ 10       │ -        │ e        │
│ 6    │ =       │ 8        │ -        │ f        │
│ 7    │ *       │ e        │ f        │ t2       │
│ 8    │ =       │ t2       │ -        │ d        │
│ 9    │ >=      │ a        │ b        │ t3       │
│ 10   │ if_not  │ t3       │ -        │ l0       │
│ 11   │ +       │ a        │ b        │ t4       │
│ 12   │ =       │ t4       │ -        │ a        │
│ 13   │ label   │ l0       │ -        │ -        │
│ 14   │ *       │ e        │ f        │ t5       │
│ 15   │ *       │ t5       │ 100      │ t6       │
│ 16   │ =       │ t6       │ -        │ g        │
│ 17   │ =       │ 10       │ -        │ u        │
│ 18   │ =       │ 99       │ -        │ j        │
└──────┴─────────┴──────────┴──────────┴──────────┘

ADVANTAGES:
  ✓ Each result is explicitly stored (arg4)
  ✓ Easy to generate code directly from quadruples
  ✓ Good for compiler backends
  ✗ More memory due to result field


2. TRIPLES FORMAT (op, arg1, arg2)
────────────────────────────────────────────────────────────────────────────

Format: Each instruction has 3 fields; result is implicit (array index)
┌──────┬─────────┬──────────┬──────────┐
│ Idx  │ Operator│ Arg1     │ Arg2     │
├──────┼─────────┼──────────┼──────────┤
│ 0    │ =       │ 10       │ -        │  → result stored at [0]
│ 1    │ =       │ 9        │ -        │  → result stored at [1]
│ 2    │ +       │ a        │ b        │  → result stored at [2]
│ 3    │ +       │ (2)      │ 100      │  → uses instruction [2], stores at [3]
│ 4    │ =       │ (3)      │ -        │  → uses instruction [3], stores at [4]
│ 5    │ =       │ 10       │ -        │  → result stored at [5]
│ 6    │ =       │ 8        │ -        │  → result stored at [6]
│ 7    │ *       │ (5)      │ (6)      │  → uses [5] and [6], stores at [7]
│ 8    │ =       │ (7)      │ -        │  → uses [7], stores at [8]
│ 9    │ >=      │ a        │ b        │  → result stored at [9]
│ 10   │ if_not  │ (9)      │ -        │  → uses [9]
│ 11   │ +       │ a        │ b        │  → result stored at [11]
│ 12   │ =       │ (11)     │ -        │  → uses [11], stores at [12]
│ 13   │ label   │ l0       │ -        │  → label reference
│ 14   │ *       │ (5)      │ (6)      │  → uses [5] and [6], stores at [14]
│ 15   │ *       │ (14)     │ 100      │  → uses [14], stores at [15]
│ 16   │ =       │ (15)     │ -        │  → uses [15], stores at [16]
│ 17   │ =       │ 10       │ -        │  → result stored at [17]
│ 18   │ =       │ 99       │ -        │  → result stored at [18]
└──────┴─────────┴──────────┴──────────┘

ADVANTAGES:
  ✓ Compact representation (3 fields vs 4)
  ✓ Implicit result location saves memory
  ✓ Common in optimizing compilers
  ✗ Arguments may reference other instructions (using indices)
  ✗ Harder to read and maintain


3. INDIRECT TRIPLES FORMAT
────────────────────────────────────────────────────────────────────────────

Format: Separate Index Pointer Table + Triples Table

INDEX POINTER TABLE (maps to triples):
┌──────────┐
│ Pointer  │
├──────────┤
│ 0        │ → points to triple 0
│ 1        │ → points to triple 1
│ 2        │ → points to triple 2
│ 3        │ → points to triple 3
│ ...      │
└──────────┘

COMBINED VIEW (Index Pointer Table + Triples):
┌──────┬──────────┬──────┬─────────┬──────────┬──────────┐
│ Ptr  │ Instr#   │ Idx  │ Op      │ Arg1     │ Arg2     │
├──────┼──────────┼──────┼─────────┼──────────┼──────────┤
│ 0    │ 0        │ 0    │ =       │ 10       │ -        │
│ 1    │ 1        │ 1    │ =       │ 9        │ -        │
│ 2    │ 2        │ 2    │ +       │ a        │ b        │
│ 3    │ 3        │ 3    │ +       │ t0       │ 100      │
│ 4    │ 4        │ 4    │ =       │ t1       │ -        │
│ 5    │ 5        │ 5    │ =       │ 10       │ -        │
│ 6    │ 6        │ 6    │ =       │ 8        │ -        │
│ 7    │ 7        │ 7    │ *       │ e        │ f        │
│ 8    │ 8        │ 8    │ =       │ t2       │ -        │
│ 9    │ 9        │ 9    │ >=      │ a        │ b        │
│ 10   │ 10       │ 10   │ if_not  │ t3       │ -        │
│ 11   │ 11       │ 11   │ +       │ a        │ b        │
│ 12   │ 12       │ 12   │ =       │ t4       │ -        │
│ 13   │ 13       │ 13   │ label   │ l0       │ -        │
│ 14   │ 14       │ 14   │ *       │ e        │ f        │
│ 15   │ 15       │ 15   │ *       │ t5       │ 100      │
│ 16   │ 16       │ 16   │ =       │ t6       │ -        │
│ 17   │ 17       │ 17   │ =       │ 10       │ -        │
│ 18   │ 18       │ 18   │ =       │ 99       │ -        │
└──────┴──────────┴──────┴─────────┴──────────┴──────────┘

ADVANTAGES:
  ✓ Allows easy code reordering without changing indices
  ✓ Index table points to actual instruction sequence
  ✓ Good for optimization passes that reorder instructions
  ✓ Makes dead code elimination easier
  ✗ Extra overhead of pointer table
  ✗ One additional lookup needed per reference

AFTER OPTIMIZATION (Constant Propagation):
┌──────┬─────────┬──────────┬──────────┬──────────┐

├──────┼─────────┼──────────┼──────────┼──────────┤

│ 4    │ =       │ t1       │ -        │ c        │

└──────┴─────────┴──────────┴──────────┴──────────┘

CONSTANTS DISCOVERED:
  a  = 10  (from instruction 0)
  b  = 9   (from instruction 1)
  e  = 10  (from instruction 5)
  f  = 8   (from instruction 6)

FOLDING RESULTS:
  10 + 9 = 19  (computed at compile-time!)
  19 + 100 = 119 (could be further folded)
  10 * 8 = 80 (computed at compile-time!)
